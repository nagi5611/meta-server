# 変更履歴: VCポート範囲の環境変数化

## 実施日
2025年2月3日

## 概要
Voice Chat (VC) システムで使用する mediasoup の UDP ポート範囲を、従来の固定値から `.env` で指定可能にした。

---

## 変更内容

### 1. 対象の仕様

- **変更前**: UDP ポート範囲はコード内で固定（10000〜10100）
- **変更後**: 環境変数 `VC_RTC_MIN_PORT` / `VC_RTC_MAX_PORT` で指定可能。未設定時は従来どおり 10000〜10100 を使用

### 2. ポート使用の前提（変更なし）

- 1人あたり最大 **2ポート**（送信用・受信用の WebRtcTransport 各1）
- マイクON＋スピーカーONで 2ポート/人、片方のみONで 1ポート/人

### 3. 変更したファイル

| ファイル | 内容 |
|----------|------|
| `server.js` | `VC_RTC_MIN_PORT` / `VC_RTC_MAX_PORT` を読み、`mediasoupConfig.worker.rtcMinPort` / `rtcMaxPort` に反映。未設定時は 10000 / 10100 をデフォルトに使用 |
| `.env` | VCポート用の環境変数例をコメントで追加（`VC_RTC_MIN_PORT`, `VC_RTC_MAX_PORT`） |
| `VC_SETUP.md` | ポート範囲がデフォルト値および環境変数で変更可能である旨を追記。本番設定例・トラブルシューティングのファイアウォール説明を環境変数対応に更新 |

### 4. 利用方法

ポート範囲を変更する場合は `.env` に以下を設定する。

```env
VC_RTC_MIN_PORT=10000
VC_RTC_MAX_PORT=10100
```

省略した場合は **10000〜10100** が使用される。  
ファイアウォールやクラウドのセキュリティグループでは、ここで指定した範囲の UDP を開放すること。

---

## 影響範囲

- 既存の `.env` に何も追加しなければ挙動は従来と同じ（10000〜10100）
- 新規にポート範囲を変えたい環境でのみ、上記2つの環境変数を設定すればよい
