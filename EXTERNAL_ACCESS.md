# 外部から HTTP でアクセスする手順

サーバーはすでに `0.0.0.0` で待ち受けているため、以下の設定でルーター経由の外部アクセスが可能です。

## 1. ルーターでポート転送（ポート開放）

| 用途 | ポート | 説明 |
|------|--------|------|
| **本番** (`npm run start:prod`) | TCP **3000** のみ | Express がすべて配信。3001 は使わない。 |
| **開発** (`npm run dev`) | TCP **3001** のみ | Vite が 3001 で配信し、API/ソケットは内部で 3000 にプロキシする。外部からは 3001 にアクセスすればよい。 |

- **転送先**: このアプリを動かしている PC の **ローカル IP**（例: 192.168.x.x）
- ルーターの管理画面で「ポート転送」「仮想サーバー」「NAT」などから上記のポートを追加する。

※ 機種によって「内部ポート」「外部ポート」を同じ番号（3000 または 3001）にすることが多いです。

## 2. Windows ファイアウォール

- 使うポート（**3000** または **3001**）への着信を許可する。
- コントロールパネル → Windows Defender ファイアウォール → 詳細設定 → 受信の規則 → 新しい規則で「ポート」を選び、TCP 3000 を許可。

## 3. 外部からのアクセス先

- **本番**: `http://<公衆IP>:3000` 例: `http://220.208.10.216:3000`
- **開発** (Vite): `http://<公衆IP>:3001` 例: `http://220.208.10.216:3001`
- ダイナミック DNS を使う場合は `http://<ドメイン>:3000` または `:3001`

### ルーターの WAN IP（公衆 IP）の調べ方

- **方法A（いちばん簡単）**: この PC のブラウザで **https://www.cman.jp/network/support/go.html** や **https://ifconfig.me** を開く。表示される IP が、今の回線の公衆 IP（＝ルーターの WAN IP）です。ポート転送を設定したあと、外部からは `http://<このIP>:3000` でアクセスできます。
- **方法B**: ルーターの管理画面（多くは `http://192.168.0.1` や `http://192.168.1.1`）にログイン → 「接続情報」「WAN」「インターネット側」などの項目で **グローバル IP** を確認する。

## 4. 音声・ビデオチャットを外部で使う場合（WebRTC）

外部から音声／ビデオも使う場合は以下が必要です。

- **.env** で `MEDIASOUP_ANNOUNCED_IP` を **公衆 IP または外部用ドメイン** に設定する（例: `220.208.10.216`）。LAN 用の 192.168.x.x のままでは外部からつながらない。
- ルーターで次の **UDP** ポート範囲を、同じくこの PC のローカル IP に転送する:
  - 10000–11000（音声 VC）
  - 20000–21000（PDF VC）
  - 30000–31000（ビデオ VC）  
  （.env の `VC_RTC_*` / `PDF_VC_RTC_*` / `VIDEO_VC_RTC_*` で変更している場合はその範囲）

## 5. 確認

- スマートフォンを **Wi‑Fi ではなくモバイル回線（4G/5G）** にして、ブラウザで `http://<公衆IP>:3000`（本番）または `:3001`（開発）を開き、画面が表示されればルーター開放は成功です。

---

## つながらない・「応答が長すぎる」ときの確認

「応答が長すぎる」は、接続がタイムアウトしている状態です。次を順に確認してください。

### ① 同じ LAN 内でまず通す（サーバーとファイアウォールの確認）

- スマホを **同じ Wi‑Fi** にして、ブラウザで `http://<PCのローカルIP>:3001` を開く。
- PC のローカル IP は、PC で **PowerShell** を開き `ipconfig` → 「IPv4 アドレス」（例: 192.168.1.10）。
- **ここでつながらない** → サーバーが動いていないか、**Windows ファイアウォール**でポートが塞がれている。  
  → ファイアウォールで **TCP 3001**（開発の場合）の「受信の規則」を追加して許可する。

### ② ルーターのポート転送の確認

- 転送先は **今の PC のローカル IP** と一致しているか（`ipconfig` で再確認）。DHCP で IP が変わっていると転送先がずれる。
- **外部ポート・内部ポート** を両方 **3001**（または 3000）にしているか。
- ルーターによっては「プロトコル」で **TCP** を選ぶ必要がある。

### ③ 公衆 IP が変わっていないか

- 多くの回線では **公衆 IP が変わります**。ルーターの「接続情報」や「WAN 側 IP」で、今の公衆 IP を確認する。
- 変わっていたら、新しい IP で `http://<新しい公衆IP>:3001` を試す。

### ④ CGNAT（キャリアグレード NAT）の可能性

- 光回線などで、**ルーターの WAN IP が「100.x.x.x」や「10.x.x.x」** の場合、プロバイダが CGNAT を使っており、自宅ルーターに直接ポート転送できません。
- その場合は **Cloudflare Tunnel** や **ngrok** など、トンネル方式で外に出す必要があります（別途設定が必要）。

### ⑤ 動作確認コマンド（PC 上）

- サーバー起動後、**同じ PC** のブラウザで `http://localhost:3001` が開くか確認する。
- 開くならサーバーは問題なし。①〜④ のネットワーク側を疑う。

### ⑥ スマホからは入れるが、同じ LAN の PC から公衆 IP で入れない

- **原因**: 多くの家庭用ルーターは **NAT ループバック（ホームルーター経由で自分の公衆 IP にアクセスすること）** に対応していません。スマホはモバイル回線なので公衆 IP に直接つながり、PC は同じ Wi‑Fi なので「自分の公衆 IP」に届かないことがあります。
- **対処**: サーバーと同じ LAN にいる **PC からは、公衆 IP ではなくローカル IP で開く**。例: `http://192.168.0.20:3000`（サーバーを動かしている PC の IPv4 アドレス）。外からだけ公衆 IP（`http://220.208.10.216:3000`）を使う。
